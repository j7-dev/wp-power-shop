## Epic 1：Dashboard 總覽

### US-01-01 查看 KPI 卡片

**使用者故事**
> 身為一個**電商網站管理者**，
> 我想要在進入後台時**立即看到關鍵業績指標**（總營收、新用戶、未出貨訂單、未付款訂單），
> 以便我能在最短時間內掌握今日商店狀況。

**驗收標準**

```gherkin
Scenario: 成功載入 KPI 卡片
  Given 管理者已登入 WordPress 後台
  And 管理者具有 manage_woocommerce 權限
  When 管理者進入 Power Shop Dashboard 頁面
  Then 系統應在 500ms 內顯示 4 張 KPI 卡片
  And 每張卡片顯示當期數值與同期比較百分比
  And 正成長顯示綠色箭頭，負成長顯示紅色箭頭

Scenario: API 請求失敗時的降級處理
  Given 管理者進入 Dashboard 頁面
  When `/power-shop/reports/dashboard/stats` API 回傳 5xx 錯誤
  Then 系統應顯示錯誤提示訊息（非空白卡片）
  And 提供「重新整理」按鈕讓管理者重試
```

### US-01-02 查看商品銷售排行榜

**使用者故事**

```gherkin
  Then 排行榜資料自動更新為選定期間的數據
  And 日期選擇器顯示所選的日期範圍
```

## Epic 2：訂單管理

### US-02-01 搜尋與篩選訂單

**使用者故事**
> 身為一個**電商客服人員**，
> 我想要能夠**快速搜尋特定訂單**或依狀態篩選，
> 以便我能即時處理顧客的問題。

**驗收標準**

```gherkin
Scenario: 關鍵字搜尋訂單
  Given 管理者在訂單列表頁面
  When 管理者在搜尋欄輸入顧客 Email「customer@example.com」後按下 Enter
  Then 列表僅顯示該顧客的訂單
  And 搜尋結果應在 300ms 內更新（前端防抖處理）

Scenario: 依狀態篩選訂單
  Given 管理者在訂單列表頁面
  When 管理者點擊「Processing（處理中）」狀態篩選
  Then 列表僅顯示狀態為 Processing 的訂單
  And 頁面標題或篩選標籤應顯示目前套用的篩選條件

Scenario: 清除篩選條件
  Given 管理者已套用狀態篩選
  When 管理者點擊「清除篩選」按鈕
  Then 列表恢復顯示所有訂單
```

## Epic 3：商品管理

### US-03-01 快速建立草稿商品

**使用者故事**
> 身為一個**電商商品管理人員**，
> 我想要**快速建立一個草稿商品**，先佔位後再慢慢填入詳細資訊，
> 以便在上架前的備稿流程更有效率。

**驗收標準**

```gherkin
Scenario: 成功建立草稿商品
  Given 管理者在商品列表頁面
  When 管理者點擊「快速建立」按鈕，輸入商品名稱「測試商品 2025」後按下 Enter
  Then 系統建立一筆狀態為 Draft 的商品
  And 新商品出現在商品列表頂端
  And 系統顯示「商品已建立為草稿」的 Toast 通知
  And 點擊商品名稱可進入商品編輯頁面

Scenario: 建立失敗（名稱為空）
  Given 管理者在商品列表頁面
  When 管理者點擊「快速建立」後未輸入名稱直接按 Enter
  Then 系統顯示驗證錯誤「商品名稱為必填」
  And 不建立任何商品
```

### US-03-02 管理商品變體

**使用者故事**
> 身為一個**電商商品管理人員**，
> 我想要**根據商品屬性自動生成所有變體**，並批量編輯各變體的價格與庫存，
> 以便管理具有多種規格的商品（如不同顏色、尺寸）。

**驗收標準**

```gherkin
Scenario: 自動生成變體
  Given 管理者在商品編輯頁的「規格」Tab 中，已設定「顏色」屬性（紅/藍）和「尺寸」屬性（S/M/L）
  And 兩個屬性均勾選「用於建立變體」
  When 管理者切換到「變體」Tab 並點擊「自動生成所有變體」
  Then 系統生成 6 個變體（2 × 3 = 6）
  And 每個變體顯示其屬性組合（如「紅色 / S」）
  And 每個變體可獨立設定價格、SKU、庫存

Scenario: 批量設定變體價格
  Given 管理者在商品編輯頁的「變體」Tab，已有 6 個變體
  When 管理者使用「批量設定」功能，將所有變體的一般售價設為 NT$ 500
  Then 所有 6 個變體的售價欄位更新為 500
  And 管理者可個別覆蓋特定變體的售價
```

```gherkin
  And 分類樹即時更新顯示新的階層關係
  And 系統自動儲存新的排序設定

Scenario: 切換至扁平列表視圖
  Given 管理者在分類管理頁面
  When 管理者點擊「列表視圖」按鈕
```

### US-03-04 設定商品庫存

**使用者故事**
> 身為一個**電商倉管人員**，
> 我想要設定每個商品的**庫存數量與缺貨處理方式**，
> 以便系統自動管控庫存並避免超賣。

**驗收標準**

```gherkin
Scenario: 啟用庫存管理
  Given 管理者在商品編輯頁的「庫存」Tab
  When 管理者勾選「啟用庫存管理」
  Then 庫存數量欄位變為可輸入
  And 缺貨處理下拉選單變為可選
  When 管理者輸入庫存數量「50」，選擇缺貨處理「不允許下單」，點擊「儲存」
  Then 商品庫存設定儲存成功
  And 當庫存歸零時，商品自動顯示「缺貨」狀態

Scenario: 停用庫存管理
  Given 管理者在商品編輯頁的「庫存」Tab，目前已啟用庫存管理
  When 管理者取消勾選「啟用庫存管理」
  Then 庫存數量欄位隱藏或變為不可輸入
  And 商品不再受庫存數量限制
```

### US-04-02 編輯 User Meta（危險操作）

> 以便修正異常的顧客資料或調整系統標記。

**驗收標準**

