## FR-01 Dashboard 總覽

### FR-01-1 KPI 卡片

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-01-1-1 | 系統應顯示**總營收**（Revenue）KPI 卡片，包含當期數值與同期比較百分比 | Must |
| FR-01-1-2 | 系統應顯示**新註冊用戶**數量 KPI 卡片，包含當期數值與同期比較百分比 | Must |
| FR-01-1-3 | 系統應顯示**未出貨訂單數**（Processing 狀態）KPI 卡片 | Must |
| FR-01-1-4 | 系統應顯示**未付款訂單數**（Pending 狀態）KPI 卡片 | Must |
| FR-01-1-5 | KPI 卡片數值應透過 `GET /power-shop/reports/dashboard/stats` 端點取得 | Must |

### FR-01-2 商品與顧客排行榜

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-01-2-1 | 系統應顯示**商品銷售排行榜**，依銷售數量或營收排序，預設顯示前 5 名 | Must |
| FR-01-2-2 | 系統應顯示**顧客消費排行榜**，依消費金額排序，預設顯示前 5 名 | Must |
| FR-01-2-3 | 排行榜應提供**日期範圍選擇器**，讓使用者自訂查詢期間 | Must |
| FR-01-2-4 | 排行榜應支援**今日、本週、本月、本年**等快速篩選選項 | Should |

### FR-01-3 營收趨勢圖表

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-01-3-1 | 系統應顯示營收趨勢**區間圖表**（Area Chart），X 軸為時間、Y 軸為金額 | Must |
| FR-01-3-2 | 圖表應支援**雙軸顯示**：左 Y 軸為營收金額，右 Y 軸為訂單數量 | Must |
| FR-01-3-3 | 使用者可選擇**日 / 週 / 月 / 年**為統計區間 | Must |
| FR-01-3-4 | 圖表應顯示**去年同期比較**曲線（灰色/虛線區分） | Should |
| FR-01-3-5 | 圖表應顯示趨勢指標（年增率 YoY%） | Should |

## FR-03 商品管理

### FR-03-1 商品列表

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-1-1 | 系統應顯示商品列表，包含：名稱、類型、SKU、庫存、售價、狀態 | Must |
| FR-03-1-2 | 商品列表應支援**分頁**（10 / 25 / 50 筆/頁） | Must |
| FR-03-1-3 | 商品列表應支援按**商品類型**篩選（Simple / Variable / Subscription / External / Grouped） | Must |
| FR-03-1-4 | 商品列表應支援按**發布狀態**篩選（Published / Draft / Trash） | Must |
| FR-03-1-5 | 商品列表應支援按**商品分類**篩選 | Should |
| FR-03-1-6 | 使用者可在列表中**快速建立草稿商品**（輸入名稱後一鍵建立） | Must |

### FR-03-2 商品編輯 — 描述 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-2-1 | 使用者可編輯商品**名稱**（必填）與 **Slug**（URL 別名） | Must |
| FR-03-2-2 | 使用者可編輯商品**簡短描述**（Rich Text） | Must |
| FR-03-2-3 | 使用者可編輯商品**完整描述**（Rich Text / Blocks） | Must |
| FR-03-2-4 | 使用者可透過 WordPress 媒體庫設定**主圖**與**相片庫** | Must |
| FR-03-2-5 | 使用者可為商品指定**分類**（多選、階層式） | Must |
| FR-03-2-6 | 使用者可為商品新增/移除**標籤** | Must |

### FR-03-3 商品編輯 — 價格 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-3-1 | 使用者可設定**一般售價**（Regular Price） | Must |
| FR-03-3-2 | 使用者可設定**特價**（Sale Price）及特價**起訖日期** | Must |
| FR-03-3-3 | 系統應支援**訂閱制定價**（需 WooCommerce Subscriptions 外掛）：訂閱週期、試用期、加入費 | Should |
| FR-03-3-4 | 外部商品類型應可設定**購買連結 URL** 與**按鈕文字** | Should |

### FR-03-4 商品編輯 — 庫存 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-4-1 | 使用者可輸入商品 **SKU**（庫存管理單位）| Must |
| FR-03-4-2 | 使用者可輸入商品 **GTIN**（全球商品識別碼） | Should |
| FR-03-4-3 | 使用者可啟用/停用**庫存管理**，並設定庫存數量與安全庫存警示值 | Must |
| FR-03-4-4 | 使用者可設定**缺貨處理方式**（不允許下單 / 允許下單 / 允許並通知） | Must |
| FR-03-4-5 | 使用者可設定每筆訂單的**購買數量限制**（最小/最大值） | Should |

### FR-03-5 商品編輯 — 規格 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-5-1 | 使用者可為商品新增/編輯/刪除**屬性**（Attribute），每個屬性包含名稱與選項值 | Must |
| FR-03-5-2 | 屬性列表支援**拖拉排序**（Drag & Drop） | Should |
| FR-03-5-3 | 使用者可從全域商品規格選擇屬性，或新增自訂屬性 | Must |
| FR-03-5-4 | 可勾選屬性是否**用於建立變體**（Use for Variations） | Must |

### FR-03-6 商品編輯 — 變體 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-6-1 | 系統可根據規格 Tab 定義的屬性**自動生成所有變體組合** | Must |
| FR-03-6-2 | 變體應以**虛擬表格**呈現，支援批量編輯（價格、庫存、SKU） | Must |
| FR-03-6-3 | 每個變體可單獨設定**價格、SKU、庫存數量、圖片** | Must |
| FR-03-6-4 | 使用者可手動新增或刪除單一變體 | Must |
| FR-03-6-5 | 使用者可**批量設定**所有變體的特定欄位值 | Should |

### FR-03-7 商品編輯 — 進階 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-7-1 | 使用者可設定**購買備註**（顯示於顧客訂單確認頁） | Should |
| FR-03-7-2 | 使用者可設定**選單順序**（Menu Order，控制商品排序） | Should |

### FR-03-8 商品編輯 — 關聯 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-8-1 | 使用者可設定商品所屬的**分組商品**（Grouped Products） | Should |
| FR-03-8-2 | 使用者可設定**追加銷售商品**（Upsells，顧客可能更感興趣的商品） | Should |
| FR-03-8-3 | 使用者可設定**交叉銷售商品**（Cross-sells，購物車推薦） | Should |

### FR-03-9 商品編輯 — 分析 Tab

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-9-1 | 系統應顯示單一商品的**銷售數量趨勢**圖表 | Should |
| FR-03-9-2 | 系統應顯示單一商品的**營收趨勢**圖表，支援日期範圍篩選 | Should |

### FR-03-10 分類管理

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-10-1 | 系統應提供**階層式分類樹**（SortableTree），支援拖拉調整父子關係與排序 | Must |
| FR-03-10-2 | 系統應提供**扁平化分類列表**（SortableList）視圖切換 | Should |
| FR-03-10-3 | 使用者可新增分類，設定名稱、Slug、父分類、描述與縮圖 | Must |
| FR-03-10-4 | 使用者可編輯或刪除現有分類 | Must |

### FR-03-11 標籤管理

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-11-1 | 使用者可新增、編輯、刪除商品**標籤** | Must |
| FR-03-11-2 | 標籤管理應顯示每個標籤的商品數量 | Should |

### FR-03-12 全域商品規格管理

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-03-12-1 | 使用者可新增全域**商品規格**（Attribute），設定名稱與選項值 | Must |
| FR-03-12-2 | 使用者可編輯與刪除現有全域規格 | Must |
| FR-03-12-3 | 全域規格可在商品編輯的規格 Tab 中被引用 | Must |

## FR-07 系統與共用功能

### FR-07-1 認證與授權

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-07-1-1 | 使用者必須擁有 WordPress **manage_woocommerce** 能力才能存取 Power Shop 後台頁面 | Must |
| FR-07-1-2 | 所有 REST API 呼叫應透過 **WP Nonce** 或 **Cookie Authentication** 進行驗證 | Must |

### FR-07-2 環境設定

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-07-2-1 | 系統應透過 **SimpleEncrypt** 加密機制將環境設定（WP REST URL、Nonce 等）傳遞至前端 | Must |
| FR-07-2-2 | 開發模式下，Vite Dev Server（port 5178）應支援 HMR（Hot Module Replacement） | Must |

### FR-07-3 Legacy 一頁賣場支援

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-07-3-1 | 系統應相容既有的 Legacy 一頁賣場功能，以載入方式支援，不主動擴充功能 | Must |

### FR-07-4 即將推出功能

| ID | 需求描述 | 優先級 |
|----|---------|--------|
| FR-07-4-1 | Marketing 模組頁面應顯示**佔位頁（Coming Soon）**，不提供實際功能 | Low |
| FR-07-4-2 | OneShop 模組頁面應顯示**佔位頁（Coming Soon）**，不提供實際功能 | Low |
